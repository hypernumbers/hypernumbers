A Quick Tour

There are a number of tutorial pages for you to get started. You can select them when you create a new page from the first tab in this dialog box. They are:

  • consolidation - instead of emailing loads of copies of a spreadsheet around and then plucking the responses from your mail box
  • negotiating - for when you need to take colleagues, suppliers or partners through a process of agreeing a set of figures
  • wikipage views - allowing people to update specific cells only
  • webpage views - one-click publishing your spreadsheets
  • forms and table views - the one-click form builder and its data/li>
  • graphs - making and publishing graphs
  • advanced web - some of our experimental web stuff

Add yourself as an administator on the views menu to make this website yours

There are also videos on hypernumbers or click the buttons below.

Just A Normal Spreadsheet

The Hypernumbers spreadsheet is pretty much a normal spreadsheet. It has:

  • a set of Excel/Open Office compatible functions
  • Excel-compatible formats
  • formula dragging
  • cut'n'paste
  • column and row resizing - use the right click menu
  • the normal keyboard short cuts

There is not really a lot to learn about the basic stuff. There is some special stuff you will need to know to get the most out of Hypernumbers though - which is covered on the next page of the tour.

Consolidating Spreadsheets

Hypernumbers is heirarchical.

On the desktop you put files in folders. So c: -> users -> johnsmith -> budgets -> 2010 might contain:

  • costcentrebudget1.xls
  • costcentrebudget2.xls

With Hypernumbers the directory structure and the spreadsheet are merged together:

  • /budgets/2010/costcentrebudget1/
  • /budgets/2010/costcentrebudget2/

But /budgets/2010/ is also a spreadsheet and it can consolidate the budgets underneath it. See the template consolidation for more details.

Understanding Views

With Hypernumbers you can have 4 different views of each spreadsheet:

  • a spreadsheet view - for people who need to change the structure of the spreadsheet
  • a webpage view - for people who need to see the contents of the spreadsheet
  • a wikipage view - for people who need to input data into a spreadsheet some else has built
  • a table view - for people who need to sort a spreadsheet

You can explore views by using the views menu at the top right of the toolbar.

You can give different people different permissions to see different views of the same page - see page 6 of the tour. The next page of this tour discusses the wikipage in more detail.

The Wikipage View

The wiki page view is just a webpage where some items can be edited.

To make a wiki page just build a normal spreadsheet and identify the cells that you want some one else to input data into.

Toggle wiki cells with the wiki button on the toolbar.

Wiki pages are designed to make negotiating and agreeing figures as easy as possible. Issue a spreadsheet with all the data you are trying to collect as wiki cells and un-wiki each line item as you agree it.

See the templates negotiating and wiki pages for details

Users And Permissions

There is one place to manage all users and their access to all your spreadsheets.

You can explore permissions by using the view menu at the top right of the toolbar. You will see little tabs for users and groups.

For quick (and dirty) sharing just set up the page as you want it, choose the default view and enter as e-mail address and click.

For proper sharing you need to do a little more work. First create a group like accounts or staff. Then add the email addresses of your people to that group. Finally add the group to the page views.

Note quick'n'dirty works by doing all these three stages in one - it turns an email into a group (eg jenny@example.com becomes the group jenny) and adds the group to the page view. The problem comes when you have jenny@example.com and jenny@hypernumbers.com!

Forms And Table Views

Forms are a neat way to collect data into a table (or database) spreadsheet.

To make a form simply select some cells, and then use Create Form on the right-click menu.

Forms can be deployed on either webpages or wikipages.

When a user fills in a form the results are posted to another page. The input is a table spreadsheet and is ideally displayed as a table view.

For more details see the template forms and tables.
